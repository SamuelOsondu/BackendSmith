# BackendSmith

> A system that makes AI build backend systems like a real engineer.

I built BackendSmith because I got tired of seeing AI generate messy backend systems.

BackendSmith forces AI coding agents to behave like real backend engineers — thinking through architecture, making proper decisions, maintaining structure, and building systems that are actually reliable.

This is something I take seriously.

If you're building backends with AI, this will change how you work.

---

## A Professional Backend Operating System for AI Coding Agents

BackendSmith turns AI coding agents into structured, disciplined backend engineers.

It is not a prompt pack.  
It is not a code generator.

It is a system that forces AI to:

- think before coding  
- make real engineering decisions  
- structure projects properly  
- maintain memory across sessions  
- build like a senior backend developer  

---

## What You Get

- Structured planning before implementation  
- Strong, production-minded architecture decisions

## Quick Start (This Is All You Need)

### 1. Add BackendSmith to your project

Copy the `skill/` folder into your project.

---

### 2. Create your project brief

Create a file like:

```

project_brief.md

```

Describe:
- what you're building  
- users  
- flows  
- features  
- integrations  

---

### 3. Open your AI coding agent

Paste this starter prompt:

```

Read the BackendSmith skill files in this project and follow them strictly.

Then read the project brief.

Start with planning only. Do not write code yet.

Follow the BackendSmith workflow:

* understand the project
* ask only necessary questions
* make strong technical decisions
* create the workplan folder
* break the system into components

Behave like a senior backend engineer.
Be concise, structured, and practical.

```

---

### 4. Let it plan

The agent will:
- analyze your system  
- ask important questions  
- propose architecture  
- create a `workplan/` folder  

---

### 5. Review briefly

Check:
- architecture  
- components  
- integrations  

Adjust if needed.

---

### 6. Start building

Then use:

```

Read BackendSmith and the workplan.

Implement the next component only.

Follow the component spec strictly.
Update the workplan after implementation.
Review your work for correctness and edge cases.

```

Repeat until done.

---

## That’s it.

No long prompts.  
No repeated explanations.  
No chaos.

---

## Important Rule

Do NOT:
- ask the agent to build everything at once  
- skip planning  
- ignore the workplan
